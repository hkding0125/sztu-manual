# 参与贡献

这本手册是开源的，能持续有用的前提是有人不断更新和补充。如果你发现信息过时了、写错了、或者你在某个方面有值得分享的经验，欢迎提交上来。

## 怎么提交

Fork [本项目仓库](https://github.com/hkding0125/sztu-manual)，在 `docs/guide/` 目录下修改或新增 Markdown 文件，然后提交 Pull Request，简单说明你改了什么、为什么改。审核后会合并到主分支。

## 关于写作

信息要准确，拿不准的标注出来。给具体的建议，把“为什么”讲清楚。保持友善，对事不对人。

如果你要补的是经验稿或访谈稿，最有用的写法通常不是“我觉得这个世界如何”，而是下面这种顺序：

- 先交代你是谁，你当时处在什么位置
- 再说你当时面临什么选择
- 接着写你为什么这么选
- 最后写结果、后果，以及如果重来一次你会怎么做

这样后来的人才能判断：这段经验到底适不适合自己，而不是只觉得你讲得很有气势。

还有一个原则：别假装自己在替所有人发言。你可以有立场，甚至可以很尖锐，但最好把“这只是我这条路上的经验”说清楚。这样你的文章会更真，也更有用。

## 一个够用的访谈 / 经验稿模板

如果你不知道怎么起笔，可以直接照这个顺序写：

1. **你是谁**
   - 专业 / 年级 / 最后去了哪
   - 如果你愿意，也可以写大概背景，比如绩点区间、科研经历、实习经历
2. **你当时面临什么选择**
   - 比如：留学还是读研，继续做材料还是转代码，先进组还是先保成绩
3. **你为什么最后这么选**
   - 写真实原因，不用写成完美答案
4. **结果怎么样**
   - 有什么收获，踩了什么坑，付了什么代价
5. **如果重来一次**
   - 你会更早做什么，不会再做什么
6. **一句留给后来人的话**
   - 不求正确，只求真

这种稿子比“我觉得大家都应该怎样怎样”有用得多。后来的人不是来听你下结论的，他们是来判断：你的这条路，对我有没有参考价值。
